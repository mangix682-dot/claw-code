# Windows 编译、配置与运行教程

> 本文档从仓库根目录 `README.md` 中抽取所有与 Windows 相关的内容，整理成一份可独立按步骤执行的指引。命令、路径、环境变量名称等技术细节保持英文原样以便直接复制。
>
> 适用平台：**Windows + PowerShell**（主路径）。Git Bash / WSL 是可选的替代方案，并非必需。

## 重要前置说明

- **不要使用** `cargo install claw-code`。`crates.io` 上的 `claw-code` 是已废弃的 stub 包，运行后只会输出 `"claw-code has been renamed to agent-code"`，且安装的二进制名为 `claw-code-deprecated.exe`，并非本仓库要构建的 `claw`。
- 上游另有一个独立发布的二进制包 `agent-code`（安装后在 Windows 上得到 `agent.exe`，**不是** `agent-code`）：
  ```powershell
  cargo install agent-code
  ```
- 本仓库（`ultraworkers/claw-code`）**只支持从源码构建**。请按下文步骤操作。
- **认证：** `claw` 需要 API key（如 `ANTHROPIC_API_KEY`、`OPENAI_API_KEY` 等），**不支持 Claude 订阅登录**。

## 1. 安装 Rust 工具链

1. 访问 <https://rustup.rs/> 下载安装程序并运行。
2. 安装结束后，**关闭并重新打开你的终端**（PowerShell / Git Bash / Windows Terminal 等），让 `PATH` 生效。
3. 验证 Rust 是否已加入 `PATH`：
   ```powershell
   cargo --version
   ```
   如果命令失败，重新打开终端，或按照 Rust 安装器输出中提示的步骤重新设置 `PATH`，再次重试。

## 2. 克隆并编译

PowerShell、Git Bash、WSL 都可以使用以下命令（路径分隔符示例使用 `/`，PowerShell 中可同样工作）：

```powershell
git clone https://github.com/ultraworkers/claw-code
cd claw-code/rust
cargo build --workspace
```

> Git Bash / WSL 是可选项。如果你偏好 bash 风格的路径（`/c/Users/you/...` 而非 `C:\Users\you\...`），可以使用随 Git for Windows 一同安装的 Git Bash。Git Bash 中出现 `MINGW64` 提示符是正常现象，**不是**安装损坏。

`cargo build --workspace` 完成后，二进制 **不会** 自动安装到系统 `PATH`，需要按下文显式查找或手动加入。

## 3. 二进制位置

`cargo build --workspace` 在 `claw-code/rust/` 目录下完成后：

| 构建模式 | Windows 路径 |
|---|---|
| Debug（默认，编译更快） | `rust/target/debug/claw.exe` |
| Release（优化版，编译更慢） | `rust/target/release/claw.exe` |

如果你执行的是 `cargo build`（未带 `--release`），二进制位于 `debug/` 目录。

## 4. 验证构建成功

在 PowerShell 中直接通过完整路径调用 `claw.exe`：

```powershell
.\rust\target\debug\claw.exe --help
.\rust\target\debug\claw.exe doctor
```

任一命令成功即说明构建可用。`claw doctor` 是首次健康检查，会校验 API key、模型可达性以及工具配置。

> **PowerShell 注意事项：** 二进制名为 `claw.exe` 而非 `claw`。可使用 `.\target\debug\claw.exe`，或者用 `cargo run -- prompt "say hello"` 跳过路径查找。

## 5. 配置 API Key

在使用前，需要在当前终端会话中设置 Anthropic API key（注意：这里需要 Anthropic API key，**不是** Claude 订阅）：

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
```

随后即可运行：

```powershell
.\rust\target\debug\claw.exe prompt "say hello"
```

> 上述 `$env:VAR = "..."` 语法只对当前 PowerShell 会话生效。如果需要持久化，请通过系统环境变量设置或在 PowerShell `$PROFILE` 中追加。

## 6. （可选）将 `claw` 加入 PATH

如果希望在任意目录直接运行 `claw` 而无需写完整路径，README 中给出的跨平台方案是用 `cargo install`：

```powershell
# 在 claw-code/rust/ 目录下执行
cargo install --path . --force

# 之后即可在任意目录执行
claw --help
```

`cargo install` 会把可执行文件放到 Cargo 的默认 bin 目录（通常是 `%USERPROFILE%\.cargo\bin\`），该目录一般已经在 `PATH` 中。

> README 中另外给出的 **Symlink (macOS/Linux)** 与 **Update shell profile (bash/zsh)** 两个方案不适用于纯 PowerShell 环境，本文不展开。如果你在 WSL / Git Bash 中工作，可参考 README 原文。

## 7. 完整快速开始（PowerShell 串联版）

```powershell
# 1. 克隆并编译
git clone https://github.com/ultraworkers/claw-code
cd claw-code/rust
cargo build --workspace

# 2. 设置 API key（仅当前会话生效）
$env:ANTHROPIC_API_KEY = "sk-ant-..."

# 3. 验证一切就绪
.\target\debug\claw.exe doctor

# 4. 跑一个 prompt
.\target\debug\claw.exe prompt "say hello"
```

## 8. 运行测试套件

验证二进制可用之后，运行 workspace 测试：

```powershell
cd rust
cargo test --workspace
```

## 9. 故障排除

- **`command not found: claw`** — 二进制位于 `rust/target/debug/claw.exe`，并未加入 `PATH`。请用完整路径 `.\rust\target\debug\claw.exe`，或按第 6 节用 `cargo install --path .` 安装到 `%USERPROFILE%\.cargo\bin\`。
- **`permission denied`** — 这是 macOS / Linux 上的常见问题（需要 `chmod +x`），Windows 上很少遇到，可忽略。
- **构建非常慢** — 默认是 debug 模式。在 `cargo build` 后追加 `--release` 切换到优化构建以获得更快的运行性能，但**编译本身会更慢**（5–10 分钟级别）。
