# `settings.json` 全部属性参考

> 本文穷举 `claw` 实际识别的 settings.json 所有顶层键与嵌套字段，标注**类型、取值范围、是否真的被运行时消费、来自哪段代码**。
>
> 适用于以下 5 个文件中的任何一个（合并顺序见 `@d:\codingnet\claw-code\ai-context\docs\002.user-config-paths.md`）：
> `~/.claw.json`、`~/.claw/settings.json`、`<repo>/.claw.json`、`<repo>/.claw/settings.json`、`<repo>/.claw/settings.local.json`。

## 总览（顶层 14 个键）

来源：`crates/runtime/src/config_validate.rs::TOP_LEVEL_FIELDS`

| 键 | 类型 | 是否被运行时消费 | 备注 |
|---|---|---|---|
| `$schema` | string | ❌ 仅元信息 | 推荐填 `"SettingsSchema"`（见 `CLAW_SETTINGS_SCHEMA_NAME` 常量） |
| `model` | string | ✅ | 默认模型名 / 别名 |
| `aliases` | object（string → string） | ✅ | 用户自定义模型别名表 |
| `hooks` | object | ✅ | 工具调用前后钩子脚本 |
| `permissions` | object | ✅ | 权限模式 + allow/deny/ask 规则 |
| `permissionMode` | string | ⚠️ **已弃用** | 用 `permissions.defaultMode` 替代 |
| `mcpServers` | object | ✅ | MCP 服务器列表 |
| `oauth` | object | ✅ | OAuth 客户端元数据（不存 token） |
| `enabledPlugins` | object | ⚠️ **已弃用** | 用 `plugins.enabled` 替代 |
| `plugins` | object | ✅ | 插件管理（启用列表 + 路径） |
| `sandbox` | object | ✅ | 沙箱隔离策略 |
| `providerFallbacks` | object | ✅ | 主 provider 失败时的回退顺序 |
| `trustedRoots` | array&lt;string&gt; | ✅ | 受信任目录列表 |
| `env` | object | ❌ **类型通过但无人读取** | 见下方"无效字段"小节 |

> ⚠️ **重要规则**：
>
> 1. **未知键 = 直接报错**（不是警告），且会列出最接近的合法键名（编辑距离 ≤ 3 的建议）。  
>    源：`crates/runtime/src/config_validate.rs::validate_object_keys`、`suggest_field`。
> 2. **类型不对 = 直接报错**（如 `"hooks": []` 会被拒绝，因为期待 object）。
> 3. **废弃键 = 通过但产生 warning**，文件仍然加载。
> 4. **`env` 字段被验证器接受但被解析器忽略** —— 写在 settings.json 里目前不会让 `claw` 设任何环境变量。要影响 `claw` 进程的环境变量，请用操作系统层面的方式（参考 `@d:\codingnet\claw-code\ai-context\docs\001.third-party-api-config.md`）。

## `model` —— 默认模型

类型：`string`

源：`crates/runtime/src/config.rs::parse_optional_model`

直接写模型名或别名，例如：

```json
{ "model": "sonnet" }
{ "model": "claude-opus-4-6" }
{ "model": "openai/gpt-4.1-mini" }
{ "model": "qwen-plus" }
```

CLI 层的优先级（`crates/rusty-claude-cli/src/main.rs::resolve_repl_model`）：
1. CLI flag `--model`（如果不是默认值）
2. 环境变量 `ANTHROPIC_MODEL`
3. settings.json 中的 `model`
4. 编译期默认 `claude-opus-4-6`

## `aliases` —— 用户自定义模型别名

类型：`object`，要求 `string → string`

源：`crates/runtime/src/config.rs::parse_optional_aliases`

```json
{
  "aliases": {
    "fast":  "claude-haiku-4-5-20251213",
    "smart": "claude-opus-4-6",
    "cheap": "grok-3-mini"
  }
}
```

别名解析时，会先查内置别名表（`opus` / `sonnet` / `haiku` / `grok` 等，见 `crates/api/src/providers/mod.rs::MODEL_REGISTRY`），再叠加用户自定义。"alias of alias" 也支持，例如 `"fast": "haiku"` 会先解到 `haiku` 再解到 `claude-haiku-4-5-20251213`。

## `hooks` —— 工具调用前后钩子

类型：`object`

源：`crates/runtime/src/config.rs::parse_optional_hooks_config_object`

字段白名单（`HOOKS_FIELDS`）：

| 字段 | 类型 | 含义 |
|---|---|---|
| `PreToolUse` | array&lt;string&gt; | 工具调用**前**执行的 shell 命令列表 |
| `PostToolUse` | array&lt;string&gt; | 工具调用**成功后**执行的 shell 命令列表 |
| `PostToolUseFailure` | array&lt;string&gt; | 工具调用**失败后**执行的 shell 命令列表 |

```json
{
  "hooks": {
    "PreToolUse":  ["echo 'tool starting'"],
    "PostToolUse": ["echo 'tool ok'"],
    "PostToolUseFailure": ["./scripts/notify-failure.sh"]
  }
}
```

每条字符串是 shell snippet（`HookRunner` 用 `/bin/sh -c` 或 Windows 等价物执行）。

## `permissions` —— 权限模式与规则

类型：`object`

源：
- `crates/runtime/src/config.rs::parse_optional_permission_mode`
- `crates/runtime/src/config.rs::parse_optional_permission_rules`

字段白名单（`PERMISSIONS_FIELDS`）：

| 字段 | 类型 | 含义 |
|---|---|---|
| `defaultMode` | string | 默认权限模式（取值见下表） |
| `allow` | array&lt;string&gt; | 始终允许的工具/操作模式 |
| `deny` | array&lt;string&gt; | 始终拒绝的工具/操作模式 |
| `ask` | array&lt;string&gt; | 需要用户确认的工具/操作模式 |

### `defaultMode` 取值（9 个别名，3 个内部模式）

源：`parse_permission_mode_label`

| 写在 settings 中的字符串 | 内部解析为 |
|---|---|
| `"default"`、`"plan"`、`"read-only"` | `ReadOnly` |
| `"acceptEdits"`、`"auto"`、`"workspace-write"` | `WorkspaceWrite` |
| `"dontAsk"`、`"danger-full-access"` | `DangerFullAccess` |

任何其他值会**报错**（"unsupported permission mode"），文件加载失败。

### 规则字符串格式

`allow` / `deny` / `ask` 中每条字符串典型格式（来自代码里的 fixture 与上游约定）：

```text
"Read"                  // 整个工具
"Edit"
"Bash(rm -rf)"          // 工具(参数子模式)
"Write(*.md)"
```

具体匹配语义由 `PermissionPolicy::authorize`（`crates/runtime/src/permissions.rs`）决定。

### 完整示例

```json
{
  "permissions": {
    "defaultMode": "workspace-write",
    "allow": ["Read", "Edit"],
    "deny": ["Bash(rm -rf *)"],
    "ask":  ["Edit(prod/*)"]
  }
}
```

## `permissionMode` —— ⚠️ 已废弃

类型：`string`

源：`crates/runtime/src/config_validate.rs::DEPRECATED_FIELDS`

写了仍然能被解析（向后兼容），但会触发 deprecation warning。**请改用** `permissions.defaultMode`。

## `mcpServers` —— MCP 服务器表

类型：`object`，键是 server 名，值是单台 server 配置对象

源：`crates/runtime/src/config.rs::parse_mcp_server_config`

每个 server 对象通过可选的 `"type"` 字段选 transport，缺省时按 `infer_mcp_server_type` 推断：含 `url` → `http`，否则 → `stdio`。

### Transport `"stdio"`（本地子进程）

| 字段 | 类型 | 必填 | 默认 |
|---|---|---|---|
| `type` | string | 否 | 推断为 `"stdio"`（无 url 时） |
| `command` | string | **是** | — |
| `args` | array&lt;string&gt; | 否 | `[]` |
| `env` | object（string→string） | 否 | `{}` |
| `toolCallTimeoutMs` | number (u64) | 否 | 无超时 |

```json
{
  "mcpServers": {
    "files": {
      "command": "uvx",
      "args": ["mcp-server-files", "--root", "."],
      "env": { "LOG_LEVEL": "info" },
      "toolCallTimeoutMs": 30000
    }
  }
}
```

### Transport `"http"` / `"sse"`（远程 HTTP）

| 字段 | 类型 | 必填 |
|---|---|---|
| `type` | string | 否，可省（含 `url` 即推断为 `http`） |
| `url` | string | **是** |
| `headers` | object（string→string） | 否 |
| `headersHelper` | string（命令路径） | 否 |
| `oauth` | object（见下） | 否 |

### Transport `"ws"`（WebSocket）

| 字段 | 类型 | 必填 |
|---|---|---|
| `type` | string | **是**（必须显式 `"ws"`） |
| `url` | string | **是**（`ws://...` 或 `wss://...`） |
| `headers` | object | 否 |
| `headersHelper` | string | 否 |

### Transport `"sdk"`（SDK 命名注入）

| 字段 | 类型 | 必填 |
|---|---|---|
| `type` | string | **是**（必须显式 `"sdk"`） |
| `name` | string | **是** |

### Transport `"claudeai-proxy"`（managed-proxy）

| 字段 | 类型 | 必填 |
|---|---|---|
| `type` | string | **是**（必须显式 `"claudeai-proxy"`） |
| `url` | string | **是** |
| `id` | string | **是** |

### Per-server `oauth` 子对象（仅 http/sse 支持）

源：`parse_optional_mcp_oauth_config`

| 字段 | 类型 |
|---|---|
| `clientId` | string |
| `callbackPort` | number (u16) |
| `authServerMetadataUrl` | string |
| `xaa` | bool |

## `oauth` —— 全局 OAuth 客户端配置（不是凭据）

类型：`object`

源：`crates/runtime/src/config.rs::parse_optional_oauth_config`

字段白名单（`OAUTH_FIELDS`）：

| 字段 | 类型 | 必填 |
|---|---|---|
| `clientId` | string | **是** |
| `authorizeUrl` | string | **是** |
| `tokenUrl` | string | **是** |
| `callbackPort` | number (u16) | 否 |
| `manualRedirectUrl` | string | 否 |
| `scopes` | array&lt;string&gt; | 否 |

> 这里只放 OAuth flow 的元数据（哪个 client、哪个 endpoint），**不**存 access token。token 由 `~/.claw/credentials.json` 维护（已被 `claw doctor` 标记为 unsupported，参见 `@d:\codingnet\claw-code\ai-context\docs\001.third-party-api-config.md`）。

## `enabledPlugins` —— ⚠️ 已废弃

类型：`object`（plugin 名 → bool）

旧风格的插件启用列表。请改用 `plugins.enabled`。两者都会被解析（兼容），但 `plugins.enabled` 优先（后写覆盖）。

## `plugins` —— 插件管理

类型：`object`

源：`crates/runtime/src/config.rs::parse_optional_plugin_config`

字段白名单（`PLUGINS_FIELDS`）：

| 字段 | 类型 | 含义 |
|---|---|---|
| `enabled` | object（string→bool） | 启用/禁用映射，键为 plugin 名 |
| `externalDirectories` | array&lt;string&gt; | 额外的 plugin 搜索目录 |
| `installRoot` | string | 用户安装的 plugin 根目录 |
| `registryPath` | string | plugin registry 文件路径 |
| `bundledRoot` | string | bundled plugin 目录覆写 |
| `maxOutputTokens` | number (u32) | 单个 plugin 工具的输出 token 上限 |

```json
{
  "plugins": {
    "enabled": {
      "example-bundled": true,
      "noisy-plugin": false
    },
    "externalDirectories": ["./vendor/plugins"],
    "maxOutputTokens": 8192
  }
}
```

## `sandbox` —— 沙箱隔离

类型：`object`

源：`crates/runtime/src/config.rs::parse_optional_sandbox_config`

字段白名单（`SANDBOX_FIELDS`）：

| 字段 | 类型 | 取值 |
|---|---|---|
| `enabled` | bool | — |
| `namespaceRestrictions` | bool | — |
| `networkIsolation` | bool | — |
| `filesystemMode` | string | `"off"` / `"workspace-only"` / `"allow-list"` |
| `allowedMounts` | array&lt;string&gt; | 当 `filesystemMode = "allow-list"` 时生效 |

### `filesystemMode` 取值

源：`parse_filesystem_mode_label`

| 字符串 | 内部 enum |
|---|---|
| `"off"` | `FilesystemIsolationMode::Off` |
| `"workspace-only"` | `FilesystemIsolationMode::WorkspaceOnly` |
| `"allow-list"` | `FilesystemIsolationMode::AllowList` |

任何其他值会**报错**。

```json
{
  "sandbox": {
    "enabled": true,
    "networkIsolation": true,
    "filesystemMode": "allow-list",
    "allowedMounts": ["logs", "tmp/cache"]
  }
}
```

## `providerFallbacks` —— Provider 回退顺序

类型：`object`

源：`crates/runtime/src/config.rs::parse_optional_provider_fallbacks`

| 字段 | 类型 |
|---|---|
| `primary` | string（provider 名或模型名） |
| `fallbacks` | array&lt;string&gt; |

```json
{
  "providerFallbacks": {
    "primary": "claude-sonnet-4-6",
    "fallbacks": ["claude-haiku-4-5-20251213", "openai/gpt-4.1-mini"]
  }
}
```

主 provider 故障（如限流、网络、模型不可用）时按 `fallbacks` 顺序重试。

## `trustedRoots` —— 受信任目录

类型：`array<string>`

源：`crates/runtime/src/config.rs::parse_optional_trusted_roots`

```json
{
  "trustedRoots": [
    "/home/me/projects/work",
    "/home/me/projects/personal"
  ]
}
```

落入此列表的目录会跳过启动时的 trust 提示。具体策略由 `crates/runtime/src/trust_resolver.rs` 决定。

## `$schema` —— JSON schema 标记

类型：`string`

源：仅在 `TOP_LEVEL_FIELDS` 白名单内，没有解析器消费。常量 `CLAW_SETTINGS_SCHEMA_NAME = "SettingsSchema"`（`crates/runtime/src/config_validate.rs:10`）。

```json
{ "$schema": "SettingsSchema" }
```

写它不会改变行为，仅用于 IDE / 编辑器的 schema 关联。

## `env` —— 类型通过但被忽略（"无效字段"）

类型：`object`

**重要警告**：`env` 出现在 `TOP_LEVEL_FIELDS` 白名单中，所以填了不会报错；但通读 `crates/runtime/src/config.rs`，**没有任何 `parse_optional_env(...)` 函数读取它**。merge 后的 `env` 对象只在测试代码里被检查，运行时既不会把它注入子进程，也不会用作 ambient 环境变量。

如果你的目标是给 `claw` 进程或它启动的工具子进程注入环境变量，参考：
- 给 `claw` 自身进程 → 通过 OS 层面（PowerShell `$env:`、`SetEnvironmentVariable`、profile 等，详见 `@d:\codingnet\claw-code\ai-context\docs\001.third-party-api-config.md`）。
- 给单个 MCP server 子进程 → 用 `mcpServers.<name>.env`（**这个**会被 stdio transport 真正传递）。
- 给 hook 脚本 → 在 hook shell snippet 里 `export VAR=...`。

## 完整示例（带注释，可作为模板）

> JSON 不允许真正的注释。下方代码块以 JSONC 风格示意；实际写到磁盘前**请删除所有 `//` 注释行**。

```jsonc
{
  "$schema": "SettingsSchema",

  // 默认模型（可被 --model / ANTHROPIC_MODEL 覆盖）
  "model": "sonnet",

  // 自定义别名
  "aliases": {
    "fast":  "haiku",
    "smart": "opus"
  },

  // 工具调用钩子
  "hooks": {
    "PreToolUse":  ["echo '[hook] tool start'"],
    "PostToolUse": ["echo '[hook] tool ok'"],
    "PostToolUseFailure": ["./scripts/notify-failure.sh"]
  },

  // 权限策略（取代旧的 permissionMode 顶层键）
  "permissions": {
    "defaultMode": "workspace-write",
    "allow": ["Read", "Edit"],
    "deny":  ["Bash(rm -rf *)"],
    "ask":   ["Edit(prod/*)"]
  },

  // MCP 服务器（混合 transport）
  "mcpServers": {
    "files": {
      "command": "uvx",
      "args": ["mcp-server-files", "--root", "."],
      "env": { "LOG_LEVEL": "info" },
      "toolCallTimeoutMs": 30000
    },
    "remote-api": {
      "type": "http",
      "url": "https://mcp.example.com/v1",
      "headers": { "X-Tenant": "acme" }
    }
  },

  // 全局 OAuth 客户端元数据（不是 token）
  "oauth": {
    "clientId":     "claw-desktop",
    "authorizeUrl": "https://auth.example.com/authorize",
    "tokenUrl":     "https://auth.example.com/token",
    "callbackPort": 8765,
    "scopes": ["read", "write"]
  },

  // 插件管理
  "plugins": {
    "enabled": {
      "example-bundled": true
    },
    "externalDirectories": ["./vendor/plugins"],
    "maxOutputTokens": 8192
  },

  // 沙箱
  "sandbox": {
    "enabled": true,
    "networkIsolation": true,
    "filesystemMode": "allow-list",
    "allowedMounts": ["logs", "tmp/cache"]
  },

  // Provider 回退
  "providerFallbacks": {
    "primary": "claude-sonnet-4-6",
    "fallbacks": ["openai/gpt-4.1-mini", "qwen-plus"]
  },

  // 受信任目录（跳过 trust 提示）
  "trustedRoots": [
    "/home/me/projects/work"
  ]
}
```

## 与 API 凭据的关系

settings.json **不接受** `apiKey`、`baseUrl`、`authToken` 等凭据字段——这些必须走环境变量，详见 `@d:\codingnet\claw-code\ai-context\docs\001.third-party-api-config.md`。

settings.json 与环境变量的职责分工：

| 维度 | 在哪里配 |
|---|---|
| 默认模型 / 别名 / 权限 / 工具钩子 / MCP / 插件 / 沙箱 / 回退 / 受信任目录 | settings.json |
| API Key / Bearer Token / Base URL / 代理 | 环境变量 |

## 验证 settings.json 是否被正确加载

```powershell
# 1. 文件被识别（doctor 输出会列出 loaded entries）
.\rust\target\debug\claw.exe doctor

# 2. 故意写一个拼错的键看错误信息
echo '{"modle":"sonnet"}' | Set-Content -Encoding UTF8 .claw\settings.json
.\rust\target\debug\claw.exe status
# 期望输出：unknown key "modle". Did you mean "model"?
```

任何 schema 错误都会在第一个错误处中断加载（不会继续合并后续文件）。
